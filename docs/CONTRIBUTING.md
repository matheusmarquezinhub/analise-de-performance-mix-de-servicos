# Guia de Contribuição

Este documento descreve como contribuir para o projeto.

## Começando

1. Faça um fork do repositório
2. Clone seu fork localmente
3. Crie uma branch para sua feature/correção (`git checkout -b feature/sua-feature`)
4. Commit suas mudanças (`git commit -am 'Add some feature'`)
5. Push para a branch (`git push origin feature/sua-feature`)
6. Abra um Pull Request

## Padrões de Código

- Use docstrings em português ou inglês (seja consistente)
- Mantenha compatibilidade com Python 3.8+
- Execute notebooks antes de fazer commits
- Atualize o README se adicionar funcionalidades

## Reportar Bugs

Use a aba "Issues" do GitHub com uma descrição clara.

# Estilo de Código

- Sigua PEP 8
- Espaçamento: 4 espaços
- Máximo 100 caracteres por linha (preferido) ou 120 (limite)

# Testes

Sempre valide seu código antes de fazer push!
